# 05 实施路线图与验收

## 1. 实施原则

按真实业务链路分期创建全新系统，先建立可信数据、可访问资料和可验证体验，再扩大协作和集成。现有系统不参与替换、并行、双写或兼容；新系统以独立代码库、空数据库和独立部署环境开始。

## 2. 阶段与退出条件

| 阶段 | 目标 | 必须交付 | 退出条件 |
| --- | --- | --- | --- |
| 0. 决策与 POC | 锁定规则、数据和技术基线 | 角色访谈、真实项目回放、MySQL/Prisma/Node POC、字段/资源分级字典、关键原型 | 立项清单签字；schema、检索、恢复和核心体验 POC 通过 |
| 1. 可信产品与资料 | 建立前台主链路和发布治理 | PIM、字段配置、方案/文件版本、预览/下载、公开/注册授权、产品/方案详情 | 一个真实产品族可从受控数据完整发布；匿名和注册旅程通过 |
| 2. 客户协作与 FAE 支持 | 按需升级到企业/项目协作 | 个人工作台、企业/项目、支持工单、需求/候选/设计包、FAE 队列 | 客户可从个人使用升级到项目；普通 FAE 可闭环一次真实支持 |
| 3. 原厂运营闭环 | 接通样品、跟单、质量、AE 和经营 | 样品/RFQ、PCN/合规、质量/验证、部门队列、驾驶舱、外部适配器 | 一条真实 Design-in 路径可跨部门追溯和回放 |
| 4. 指标驱动扩展 | 用真实数据决定扩展能力 | 搜索增强、可选文档转换、AI 受控辅助、实验与分析 | 指标证明扩展能力提升客户或原厂结果 |

## 3. 阶段 0 的硬性 POC

阶段 0 可创建隔离的验证代码、临时 schema、测试容器和交互原型，用于验证下表指标。POC 必须可重复执行、可整体清除、不得连接或替换生产环境，也不得被标记为阶段 1 正式交付；只有 POC 证据、A-D 签字项和管理层书面确认全部完成后，才能保留或重写其中经评审通过的技术方案并进入正式开发。

| POC | 通过标准 |
| --- | --- |
| MySQL 9.7 与 Prisma | 新 schema、主外键、JSON、事务、中文全文和关键查询通过 |
| Node 24 | 构建、单测、文件预览、鉴权、Worker 在新运行时通过 |
| 干净初始化 | 最终 MySQL schema、管理员/RBAC 初始化和演示数据缺失状态均可重复验证 |
| 文件授权 | 匿名前三页、注册完整预览/下载、受限拒绝、过期令牌和管理员覆盖全部可测 |
| 体验原型 | 客户独立找方案/资料、普通 FAE 支持、系统管理员配置/覆盖均能完成 |
| 恢复 | MySQL 与对象存储从备份恢复成功，时间和数据差异有记录 |

任一 POC 不通过，只能修复或调整设计，不得进入新系统正式开发。

## 4. 关键验收

### 业务验收

- 匿名用户可找型号、方案、资料和快速选型，且公开资料前三页预览可用。
- 注册个人可保存选型、收藏、订阅、完整预览/下载开放资料，无项目强制入口。
- 产品页展示上架关联方案；方案关联时下架产品不可选。
- 产品/资料字段可由管理员新建、修改、调整类型和展示规则；旧值兼容性可见。
- 产品规格书可被关联；方案资料支持单份与全包下载；授权和版本正确。
- 客户项目、FAE 支持、样品/跟单、质量/AE 协作按阶段上线并可追溯。
- 系统管理员可操作全部资源和配置，高风险覆盖留有完整审计。

### 非功能验收

| 维度 | 最低目标 |
| --- | --- |
| 正确性 | 核心产品、方案、文件、授权、发布与字段配置有 API/组件/E2E 覆盖 |
| 性能 | 常用产品/方案页 P95 小于 1.5 秒；选型 P95 小于 2 秒；预览任务可观测、可重试 |
| 安全 | 组织/项目隔离、下载/导出授权、过期令牌、审计和管理员覆盖自动化测试通过 |
| 可靠性 | Outbox 幂等/重放、发布回滚、备份恢复演练通过 |
| 可用性 | 关键任务键盘可完成；320 CSS px 与 200% 缩放无阻塞性遮挡 |
| 数据质量 | 目标产品族关键参数完整率不低于 98%，受控资料版本覆盖率 100% |

## 5. 指标与风险控制

北极星指标是“已验证有效 Design-in 项目及其量产转化”，而不是 PV。客户价值看找方案/资料成功率、选型完成时间和资料采用率；原厂效率看 FAE 首响/结案、方案复用和发布周期；质量风险看过期资料、PCN 确认、参数完整和异常下载。

最大风险是主数据无人负责、范围失控、外部数据被当成原厂事实、平台比邮件增加步骤、内容继续私聊发送和 AI 无证据输出。每项风险必须有业务责任人、量化信号和处理期限。
