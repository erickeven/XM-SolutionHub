# 03 体验与 UI 设计系统

## 1. 设计目标

界面应传达精密仪器感、工程证据感和工业级可靠性，而不是通用后台、营销落地页或堆叠卡片。用户进入后先看到正在解决的任务、可信数据和下一步，不先看到品牌口号或配置入口。

设计借鉴成熟科技企业的共同方法：清晰信息层级、稳定栅格、数据表格可用性、无障碍和响应式，而不复制任何品牌的外观。

## 2. 三种界面

| 界面 | 用户 | 设计重点 |
| --- | --- | --- |
| 外部发现 | 匿名访客、注册个人 | 搜索和选型优先；产品、方案、资料证据可扫描；不强制开案 |
| 客户个人/协作 | 注册用户、客户企业成员、客户项目成员 | 最近选型、收藏、订阅、项目协作和受控资料，个人上下文与项目上下文清晰切换 |
| 原厂作业台 | 销售、FAE、样品/跟单、质量、AE、管理层、管理员 | 高密度队列、责任人、SLA、时间线和可处置风险 |

三种界面共享品牌令牌和基础组件，但不能共用同一导航、首屏和信息密度。

## 3. 视觉与组件规则

- 使用深海军蓝、芯片铜金、工业灰作为品牌锚点；深色只用于导航、代码/波形预览和重点区域，不能铺满页面。
- 采用 4px 间距栅格，外部内容最大宽度 1280px，内部桌面采用 12 列栅格；普通圆角 4-6px，卡片/弹窗不超过 8px。
- 颜色必须是语义令牌：`surface`、`text`、`border`、`brand`、`success`、`warning`、`danger`、`info`。业务页面不得散落色值、间距和阴影。
- Ant Design 只作为复杂表格、日期、筛选和无障碍控件的实现基础；Tailwind 不得再独立定义第二套视觉规则。
- 图标按钮必须有 tooltip、无障碍名称、焦点态和禁用原因。状态不能只依赖颜色。
- 加载、空、错误、无权四态为每个页面/组件的必需状态。

## 4. 导航与关键页面

### 外部端

首屏提供平等入口：`找型号`、`找方案`、`找资料`、`快速选型`。注册后默认进入“我的选型与资料”，而非空项目页。企业和项目切换器只在用户确实加入对应上下文后显示。

产品详情固定展示料号、状态、关键参数、收藏/比较/咨询入口；主内容按适用方案、参数、资料、生命周期/合规和替代关系组织。方案是产品详情的主区块，而不是页尾链接。

方案详情固定展示版本、适用产品、适用条件、性能/测试摘要和资料目录。文件项显示版本、语言、资源等级、有效期和预览/下载状态；用户可以下载单个资料或当前有权版本的全包。

### 原厂内部端

采用固定壳层：左侧一级任务导航、顶部全局搜索与待办、页面上下文头、主体数据区、可选右侧详情面板。各部门的首屏必须是行动队列：

| 工作台 | 首屏内容 |
| --- | --- |
| 销售经营 | 我的客户、下一步、停滞商机、FAE 支持状态 |
| FAE 技术 | 待响应问题、当前项目、待验证方案、SLA 风险 |
| 样品/跟单 | 待审批、物流/交期异常、待客户确认 |
| 质量/AE | 质量事件、测试验证任务、待发布报告 |
| 管理驾驶舱 | 指标、风险、资源负载及可下钻明细 |
| 系统治理 | 安全告警、授权变更、集成健康、审计查询 |

## 5. 内容与字段配置控制台

管理员在一个版本化控制台管理页面、导航、文案、字段定义、字段分组、可见性、资源等级和发布计划。控制台必须提供：草稿、预览、审批/管理员覆盖、定时发布、回滚、变更说明和影响分析。

产品/资料字段的新增、修改和类型变更通过字段定义生效；表单、筛选、详情展示和导入校验读取同一字段定义。已发布字段的类型变更必须检查已有业务值兼容性并提供明确阻断提示。

## 6. 体验验收

1. 匿名客户可在不注册的情况下完成找方案、查看适用条件和预览公开资料前三页。
2. 注册个人可在三次以内完成搜索、收藏、完整预览或下载开放资料，且不被迫创建项目。
3. 普通 FAE 不离开当前队列即可获取客户需求、已读资料、关联方案和待办，并提交可追溯结论。
4. 系统管理员可执行全局覆盖；高风险覆盖会生成完整审计记录。
5. 关键任务支持键盘操作；320 CSS px 宽度和 200% 缩放下没有横向遮挡；对比度达到 WCAG 2.1 AA。
