# 芯茂微数字化 Design-in 平台全新重写资料包

> 状态：最终定型，2026-07-11。  
> 用途：下一阶段从零全新开发的唯一业务、体验、技术与验收基线。  
> 说明：本目录已清理调研过程、重复评审和历史草案；后续变更必须更新对应最终文档，不再新增平行方案。

## 最终决策摘要

| 项目 | 最终结论 |
| --- | --- |
| 平台定位 | 客户技术发现与芯茂微 Design-in 协同平台，不是普通资料站、CRM、ERP 或商城替代品 |
| 客户入口 | 发现优先：匿名即可找型号、方案、资料和预览公开资料前三页；项目只在协作/样品/RFQ/专项支持时创建 |
| 角色 | 客户 FAE/工程师、采购、PM 是客户任务视图；芯茂微内部角色按业务、工程、客户管理、质量、AE 和管理层独立作业 |
| 系统管理员 | 拥有全系统任意权限；高风险覆盖必须 `break-glass` 留痕、通知和可恢复 |
| 内容与字段 | 管理员可配置全部页面、文案、导航、字段和展示；配置版本化、可预览、可发布和可回滚，领域规则不被自由文案覆盖 |
| UI | 外部发现、客户协作、原厂作业台三种界面，共享设计系统但不共用页面组织方式 |
| 核心技术 | React 19/Vite、Node 24 LTS/Express、Prisma、MySQL 9.7 LTS、Redis、SeaweedFS 4.29（S3 Adapter）、Worker、Docker Compose |
| 检索与文件 | MySQL 全文/关系检索为默认；向量搜索、Office 转换、公开门户 SSR 按量化需求启用，不作为首阶段负担 |
| 全新起点 | 现有项目的代码、数据库、数据、配置、镜像和部署均不保留、不迁移、不复用；新系统从空 MySQL 和全新仓库结构初始化 |

## 阅读与执行顺序

1. [00-总体决策与范围](00-总体决策与范围.md)：确认平台边界、主链路、首阶段范围和不做事项。
2. [01-角色权限与授权模型](01-角色权限与授权模型.md)：确认客户/原厂角色、资源授权与管理员全权限。
3. [02-业务能力与客户协同](02-业务能力与客户协同.md)：确认领域对象、协作升级、资料与字段规则。
4. [03-体验与 UI 设计系统](03-体验与UI设计系统.md)：确认三类界面、页面规则、内容配置与可用性标准。
5. [04-技术架构与数据治理](04-技术架构与数据治理.md)：确认技术栈、MySQL、Worker、文件、检索、部署和安全边界。
6. [05-实施路线图与验收](05-实施路线图与验收.md)：执行分期、POC、验收指标和风险控制。
7. [06-立项与验证清单](06-立项与验证清单.md)：在进入编码前完成签字、证据和真实任务验证。
8. [正式交付验证证据](正式交付验证证据.md)：记录最终代码、浏览器、镜像、恢复和部署验收结果。

## 使用规则

1. 新系统的代码、数据库、UI、测试、镜像和部署均从零创建，以本目录为唯一设计输入。
2. 发现规则冲突时，先修订本资料包并通过业务、技术、质量/安全评审，再改代码。
3. 未通过阶段 0 POC 和立项清单，不得启动新系统正式开发或部署切换。
4. 这套资料定义的是最终目标与分期边界，不承诺在未经 POC 的情况下启用可选组件。
