# 02 业务能力与客户协同

## 1. 客户旅程

### 轻量发现：默认路径

匿名访客或注册个人可以按应用、拓扑、参数、型号或资料标题进入系统。系统先回答“哪个产品/方案适合、证据是什么、下一步能做什么”，而不是先收集项目或采购信息。

| 页面 | 核心对象 | 必须提供 |
| --- | --- | --- |
| 找型号/快速选型 | 产品、可订货料号、参数、候选 | 硬约束、权衡、证据、比较与保存 |
| 找方案/热门应用 | 方案、适用产品、测试证据 | 按方案筛选、适用条件、版本和资料完整度 |
| 找资料 | 文件版本、方案/产品关系、资源等级 | 关键词检索、前三页预览、授权说明 |
| 产品详情 | 产品、可订货料号、关联方案 | 关键参数、生命周期、方案、规格书和支持入口 |
| 方案详情 | 方案版本、BOM、测试、资料 | 适用条件、测试环境、资料目录、单份/全包下载 |

选型结果不能只显示一个黑箱评分。每个候选必须给出硬约束满足情况、适用方案、参数来源、版本、已知限制和下一步。

### 协作升级：按客户主动意图发生

| 客户触发 | 平台动作 | 原厂承接 |
| --- | --- | --- |
| 收藏、订阅、下载开放资料 | 个人工作台记录；仅形成合规行为信号 | 不自动建商机或项目 |
| 技术咨询 | 创建支持工单或咨询记录 | 销售/FAE 分流并承诺响应时间 |
| 共享需求、专项资料授权 | 邀请企业成员或授权企业空间 | 销售/FAE 确认客户关系和授权 |
| 样品、RFQ、专项验证、正式开案 | 经客户确认创建或关联项目 | 销售、FAE、样品/跟单、AE/质量按职责协同 |

正式项目冻结需求版本、候选、方案/文件版本、关键结论、责任人和阶段。没有客户确认或业务承接，不得把浏览行为升级为项目事实。

## 2. 原厂协同链路

```text
销售接触/客户需求
-> FAE 技术支持与受控结论
-> 需要时 AE 测试验证 / 质量支持
-> 样品助理与跟单助理处理履约节点
-> 销售和客户确认 Design-in、量产或关闭
-> 管理层查看经营与风险；系统管理员治理系统和权限
```

每次交接都以项目、工单、样品、质量事件或发布包等受控对象完成，不能以邮件附件或 Excel 副本成为唯一事实源。

## 3. 领域能力与对象边界

| 能力域 | 关键对象 | 关键规则 |
| --- | --- | --- |
| 产品 PIM | 产品族、产品、可订货料号、封装、参数、认证、替代 | 产品与可订货料号分开建模；参数有单位、条件、来源与审核状态 |
| 方案资产 | 方案、方案版本、BOM、测试、文件、适配关系 | 一个方案版本可关联多个产品；资料与方案/产品关系显式保存 |
| 选型 | 需求、约束、候选、比较快照、解释 | 硬约束优先；结果版本化、可回放、可保存 |
| 资料与发布 | 文件版本、预览派生物、发布包、资源等级、下载事件 | 原件不可就地覆盖；发布包冻结版本；访问短时授权 |
| 客户协同 | 个人工作台、企业、项目、成员、工单、时间线 | 个人与企业/项目隔离；项目只在明确协作时创建 |
| 供应与质量 | 生命周期、PCN/PDN、合规、样品、RFQ、质量事件 | 外部事实标明来源/更新时间；PCN 影响与确认可追踪 |
| 内容与配置 | 页面、区块、文案、字段定义、菜单、配置版本 | 管理员可配置和回滚；业务事实与流程状态不由自由文案覆盖 |
| 分析与 AI | 规范事件、指标、受控知识、反馈 | AI 只引用当前用户有权访问的已发布证据 |

## 4. 数据与状态原则

1. 产品、料号、方案、文件、BOM、测试、发布包和项目都有稳定 ID；内容变更创建版本。
2. `ParameterDefinition` 与 `ParameterValue` 分离。字段定义决定类型、单位、校验、可见性和排序，字段值不得由页面写死。
3. `SolutionProductFit`、`BomLine`、`DocumentVersion`、`ProjectCandidate` 等关系必须使用受约束关系表，不能把关键关系塞入 JSON。
4. 资料发布状态、方案状态和项目阶段分开管理；“已上架/已发布”是前台可见性的必要条件。
5. 方案关联产品时，仅可选择上架/已发布且符合适配规则的产品；下架产品不得作为可关联候选。
6. 所有关键事实记录来源、责任人、审核状态、生效时间和失效时间，支持历史回放。

## 5. 文件预览与下载规则

| 身份与资料等级 | 预览 | 下载 |
| --- | --- | --- |
| 匿名 + PUBLIC | 前三页 | 不提供，或由资料规则明确开放 |
| 注册个人 + REGISTERED | 完整预览 | 可下载 |
| 企业/项目/NDA 成员 + 对应授权 | 完整预览 | 可下载，记录审计 |
| 无授权用户 | 不泄露标题外的受限内容；可显示申请入口 | 不可下载 |

DOCX、XLSX、PPTX 等统一产生受控 PDF 预览派生物。预览和下载都在服务端实时评估授权，不能复用过期令牌，也不能将对象存储永久链接暴露给浏览器。

## 6. 最小可交付能力

第一阶段只以一个真实产品族和一条真实客户 Design-in 路径验收。先完成可信产品/方案/资料、轻量发现、个人保存、授权下载、原厂支持工单和最小内部队列，再扩大到供应、质量、AE 和经营驾驶舱。
