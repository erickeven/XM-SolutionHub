## 2. 产品目标

芯茂微需要一个面向工程师、采购、销售与 FAE 的在线技术营销平台，将产品选型、方案资料、AI 技术问答和线索运营合并为一个闭环。

核心目标：

1. 让访客在 2 秒内获得可解释的芯片推荐。
2. 让客户无需销售介入即可预览规格书和 DEMO 报告前 3 页。
3. 通过注册解锁完整资料，形成可追踪销售线索。
4. 让注册用户可基于企业资料进行 AI 问答，回答必须带来源。
5. 让内部人员能维护产品、方案、资料、知识库、线索和权限。

## 3. 成功指标

| 指标 | 验收口径 | 目标 |
| --- | --- | --- |
| 选型接口响应 | `/api/v1/selection/match` P95 | < 2 秒 |
| AI 首字响应 | 有命中文档时的流式首字 P95 | < 3 秒 |
| PDF 首屏预览 | 首次渲染第 1 页 P95 | < 1.5 秒 |

| 来源覆盖率 | AI 有效回答中包含来源的比例 | 100% |

## 4. 范围边界

### 4.1 第一版必须实现

1. 邮箱注册、登录、退出、密码重置。
2. 产品选型：参数输入、精确匹配、近似匹配、差异说明、兜底推荐。
3. 产品详情：关键参数、优势、规格书、关联方案。
4. 资料中心：规格书预览、DEMO 报告前 3 页预览、注册后完整预览与下载。
5. 文件权限：匿名仅预览允许页数，下载必须登录，下载记录审计。
6. AI 问答：SAG 结构化检索、来源标注、无资料时拒答、反馈、历史记录。
7. 线索管理：行为采集、线索聚合、状态流转、分配、导出。
8. 运营后台：产品、方案、资料、知识库、用户、角色、线索管理。
9. 响应式体验：PC、平板、手机均可完成核心路径。

### 4.2 第一版不实现

1. 原生 App。
2. 在线人工客服。
3. 支付、订单、库存、供应商协同。
4. 多语言。
5. 复杂营销玩法。
6. 独立 BI 大屏。

## 6. 核心业务流程

### 6.1 外部客户主路径

```text
访问首页
  -> 输入应用参数
  -> 查看推荐型号和匹配理由
  -> 进入产品详情
  -> 预览规格书或 DEMO 报告前 3 页
  -> 触发第 4 页/下载拦截
  -> 注册或登录
  -> 解锁完整资料
  -> 使用 AI 问答或下载方案包
  -> 系统生成线索并进入后台跟进
```

### 6.2 内部运营路径

```text
登录后台
  -> 维护产品和参数
  -> 上传方案资料与知识库
  -> 查看线索列表
  -> 分配给销售或 FAE
  -> 更新跟进状态
  -> 导出线索和审计记录
```

## 7. 功能需求

### 7.1 产品选型

用户输入电气、性能、环境、合规、应用参数后，系统返回推荐产品。

#### 参数要求

| 参数 | 类型 | 必填 | 示例 |
| --- | --- | --- | --- |
| 输入电压最小值 | number | 是 | 85 |
| 输入电压最大值 | number | 是 | 265 |
| 输出电压 | number | 是 | 12 |
| 输出电流 | number | 是 | 2 |
| 应用类型 | enum/string | 是 | 适配器 |
| 能效等级 | enum | 否 | 六级 |
| 待机功耗上限 | number | 否 | 0.075 |
| 最大环境温度 | number | 否 | 85 |
| PCBA 尺寸 | object | 否 | `{ "width": 30, "height": 20 }` |
| 认证需求 | string[] | 否 | `["CE", "FCC"]` |
| PFC 需求 | boolean | 否 | false |

#### 匹配规则

1. 精确匹配：产品参数覆盖用户输入约束，标记为 `exact`。
2. 近似匹配：无完全覆盖或部分关键参数偏离时，计算差异分并标记为 `approximate`。
3. 兜底推荐：没有可用结果时，返回相似度最高的 3 个型号，必须展示不匹配原因。
4. 排序：精确匹配优先；同类按综合分降序；同分时优先推荐资料完整度高的产品。
5. 解释：每个结果必须包含 `matchLevel`、`score`、`reasons`、`diffs`。

#### 验收标准

- 输入完整标准参数时，2 秒内返回推荐列表。
- 无精确匹配时，返回近似列表并标出每个差异点。
- 首页快速选型至少提交输入电压范围、输出电压、输出电流和应用类型；完整选型接口缺少任一必填参数时返回参数错误。
- 用户主动清空全部参数时不调用匹配接口，前端展示热门型号并提示补充参数；热门型号不是匹配结果，不显示匹配分。
- 自定义输入值必须经过前后端双重校验。
- 推荐结果不能只给型号，必须给出推荐理由和可查看资料入口。

### 7.2 产品详情

详情页展示型号、系列、关键参数、典型应用、优势、封装、状态、规格书和关联方案。

验收标准：

- 从推荐列表进入详情页不丢失选型上下文。
- 产品下架后对外不可作为推荐结果，但后台仍可查看历史记录。
- 参数单位必须统一展示，例如 `V`、`A`、`W`、`mm`、`°C`。

### 7.3 DEMO 资料与文件权限

资料类型包括测试报告、原理图、BOM、应用指南、设计指南、规格书。

| 用户状态 | 规格书 | DEMO 报告 | 其他资料 | 下载 |
| --- | --- | --- | --- | --- |
| 匿名访客 | 可完整预览 | 仅前 3 页 | 仅展示标题和摘要 | 否 |
| 注册用户 | 可完整预览 | 可完整预览 | 可查看 | 是 |
| 内部角色 | 可完整预览 | 可完整预览 | 可查看 | 是 |

业务规则：

1. 匿名用户访问 DEMO 报告第 4 页时展示解锁弹窗。
2. 匿名用户点击下载时直接展示登录/注册弹窗。
3. 注册后回到原页面和原资料位置。
4. 下载链接必须短时有效，默认 10 分钟过期。
5. PDF 下载需增加用户标识水印；非 PDF 文件需记录下载审计。
6. 匿名预览必须读取独立生成的前 3 页派生文件，严禁向前端签发原始 PDF 地址或仅依赖前端遮罩限制页码。
7. 原文件、匿名预览件和带水印下载件使用不同 `storageKey`；临时水印文件到期后自动清理。

验收标准：

- 匿名用户无法通过 URL 参数越权访问第 4 页之后内容。
- 登录后刷新或回跳能继续查看完整资料。
- 每次下载记录用户、文件、IP、时间和来源页面。
- 资料缺失时展示“资料整理中”，不得出现空白页或死链。
- 匿名预览请求即使篡改页码、Range 请求或复制临时地址，也不能读取第 4 页后的内容。

### 7.4 AI 技术问答

AI 问答只面向登录用户开放，必须基于知识库回答。第一版知识库采用 SAG 思路：每个文档切片提取一个完整事项 `event` 和多个实体 `entity`，通过 SQL/pgvector/全文检索在查询时动态扩展关联证据，避免单纯向量 RAG 在多跳问题、结构约束和证据完整性上的缺陷。

业务规则：

1. 回答必须包含至少 1 条来源，来源含文档名、页码或段落。
2. 检索置信度不足时，回复“暂无相关资料支持该问题”，不得编造。
3. 支持流式输出，但完整回答和来源需落库。
4. 用户可对回答标记“有帮助”或“无帮助”。
5. 管理员可查看问题、回答、来源、反馈、命中 event/entity 和检索链路。
6. 极速模式默认启用：先用实体精确/前缀/trigram、正文全文排名与向量检索召回候选，再做 SQL 多跳扩展和重排。
7. 标准模式作为可配置选项：允许调用 LLM 抽取 query entities，再执行多路召回和精排。
8. 普通向量召回仅作为降级策略，不作为第一版默认检索路径。
9. 知识库文档状态统一为 `UPLOADED -> PROCESSING -> READY`，失败进入 `FAILED` 并保存可读错误；重建索引必须生成新版本，成功后原子切换，不能让半成品参与检索。
10. 来源仅指向当前用户有权访问的资料；外部用户不得通过 AI 引用读取内部或已下架内容。

验收标准：

- 有命中文档时首字响应 P95 < 3 秒。
- 无命中文档时不调用或不输出自由编造答案。
- 来源链接可跳转到对应资料或后台文档片段。
- 历史对话仅用户本人和管理员可查看。
- 被识别为多跳问题时，必须返回至少 2 个相关 event 支撑的证据链；简单事实问题允许由 1 个高置信 event 回答。证据不足时统一降级为“资料不足”。
- 后台可查看文档切片、event、entity、embedding 状态和最近一次检索 trace。
- 同一文档重复提交索引任务不得生成重复的 chunk/event/entity；失败任务可安全重试。

### 7.5 线索管理

线索来源包括注册、选型、资料预览、资料下载、AI 问答反馈。

#### 线索聚合规则

1. 登录用户按 `userId` 聚合。
2. 未登录访客按匿名设备 ID 聚合，注册后合并到用户线索。
3. 关键行为增加线索热度分：下载资料 > 注册 > AI 提问 > 方案预览 > 选型查询。
4. 同一用户保留完整行为明细，不覆盖历史。

#### 状态流转

```text
新建 -> 已分配 -> 跟进中 -> 已转化
                    -> 已放弃
```

验收标准：

- 注册成功后 5 秒内生成或合并线索。
- 审核员和管理员可筛选、分配、更新状态、导出。
- 内部用户只能查看分配给自己的线索、行为明细并更新跟进状态，不能改派、导出或访问他人线索。
- 分配和状态变更必须写入审计日志。
- 外部用户不能访问线索接口。

### 7.6 认证与账号

1. 注册和登录使用邮箱 + 密码。
2. 密码至少 8 位，包含大小写字母和数字。
3. 密码使用 bcrypt 或同等级算法加盐哈希。
4. Access Token 有效期 2 小时，Refresh Token 有效期 7 天。
5. 连续 5 次登录失败锁定 15 分钟。
6. 注册必须勾选隐私政策。
7. 密码重置链接有效期 1 小时，使用后立即失效。
8. 修改密码、禁用账号或检测到 Refresh Token 重放后，撤销该用户全部有效 Refresh Token。

## 10. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 页面 LCP P75 < 2.5 秒，核心接口有 P95 指标 |
| 安全 | HTTPS、RBAC、限流、输入校验、文件鉴权、防越权 |
| 可用性 | 关键路径出现错误时必须有可理解提示和重试入口 |
| 可观测性 | 请求日志、错误日志、审计日志、关键指标埋点 |
| 数据合规 | 用户同意隐私政策后采集个人信息，日志脱敏 |
| 备份 | 数据库每日备份、文件存储按业务策略备份；上线前至少完成一次空环境恢复演练 |
| 兼容性 | 支持当前及前一个主要版本的 Chrome、Edge、Safari；移动端覆盖 iOS Safari 与 Android Chrome |

| Phase 3 | 资料闭环 | 方案资料、PDF 预览、注册解锁、下载审计 | 匿名和登录权限符合规则 |
| Phase 4 | AI 问答 | SAG 文档索引、问答、来源、反馈、检索 trace | 有来源回答，低置信拒答 |

| Phase 6 | 上线加固 | 压测、安全检查、响应式验收、部署 | 通过上线检查清单 |

## 13. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 产品参数不标准 | 上线前建立参数字典，导入时强校验 |
| 资料缺失或涉密 | 资料状态分为草稿、上架、下架；默认不上架 |
| AI 幻觉 | 强制来源、置信度阈值、拒答策略、反馈闭环 |
| SAG 融合复杂度 | 只吸收事件/实体索引与 SQL 多跳检索，避免直接嵌入完整工作台 |
| 文件越权 | 所有预览和下载都走后端签权，不暴露永久直链 |
| UI 模板感强 | 按 `design.md` 执行页面级验收和真实数据走查 |
