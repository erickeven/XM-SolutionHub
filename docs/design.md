# 芯茂微产品选型与 DEMO 资料系统 UI/UX 设计规范

## 1. 设计定位

本系统不是普通官网，也不是电商列表页，而是面向工程师、采购和销售支持人员的技术型工作台。视觉需要体现芯片原厂的专业度、可信度和效率感，同时避免常见 AI 生成页面的空泛大标题、低密度卡片堆叠、过度渐变和装饰性插画。

设计关键词：

- 精密
- 克制
- 高级
- 信息清晰
- 技术可信
- 操作高效

第一屏必须让用户立即知道：这是芯茂微的产品选型与方案资料系统，可以直接输入参数获得芯片推荐。

## 2. 体验原则

1. 工具优先：首屏直接提供选型入口，不做纯营销落地页。
2. 信息有层级：型号、关键电气参数、匹配原因和资料入口永远优先。
3. 视觉克制：不用大面积渐变、漂浮圆球、廉价科技线条和无意义插画。
4. 数据真实：页面必须用真实字段结构设计，不能只放“标题 + 描述 + 按钮”。
5. 移动端完整：手机端也必须能选型、预览资料、登录解锁和提问。
6. 状态完整：加载、空数据、错误、无权限、下架、资料缺失都要有设计。
7. 品牌真实：首屏和产品页优先使用芯片实物、封装、PCB 或测试场景的真实素材；素材缺失时使用干净的产品占位图，不用泛化 AI 科技插画替代产品。

## 3. 视觉系统

### 3.1 色彩

主视觉采用深海军蓝、芯片铜金、工业灰和少量高亮蓝。整体气质应偏高端技术仪器，而不是蓝白 SaaS 模板。

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--color-navy-950` | `#07111F` | 顶部导航、深色强调区 |
| `--color-navy-900` | `#0B1F36` | Hero 背景、后台侧栏 |
| `--color-blue-600` | `#2563EB` | 主按钮、链接、选中态 |
| `--color-blue-100` | `#DBEAFE` | 轻量选中背景 |
| `--color-copper-500` | `#B7791F` | 高价值标签、芯片工艺强调 |
| `--color-green-600` | `#16A34A` | 精确匹配、成功 |
| `--color-amber-600` | `#D97706` | 近似匹配、警告 |
| `--color-red-600` | `#DC2626` | 错误、危险操作 |
| `--color-slate-900` | `#111827` | 一级文本 |
| `--color-slate-700` | `#374151` | 正文 |
| `--color-slate-500` | `#6B7280` | 辅助文本 |
| `--color-slate-200` | `#E5E7EB` | 边框 |
| `--color-slate-50` | `#F8FAFC` | 页面背景 |
| `--color-white` | `#FFFFFF` | 内容底色 |

使用规则：

1. 主色不超过页面可视面积的 20%。
2. 深色区域只用于首屏、导航、后台侧栏或关键对比，不铺满所有页面。
3. 精确匹配用绿色，近似匹配用琥珀色，AI 标识用蓝紫小面积点缀。
4. 禁止使用大面积紫蓝渐变作为背景。
5. 卡片不靠重阴影制造层次，优先用边框、分区、留白和字号层级。

### 3.2 字体

```css
font-family: Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
  "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

| 场景 | PC | 移动端 | 字重 |
| --- | ---: | ---: | ---: |
| 首屏标题 | 40px | 28px | 700 |
| 页面标题 | 28px | 22px | 700 |
| 分区标题 | 20px | 18px | 600 |
| 卡片标题/型号 | 18px | 16px | 700 |
| 正文 | 14px | 14px | 400 |
| 辅助文本 | 12px | 12px | 400 |
| 参数数值 | 18px | 16px | 600 |
| 表格文本 | 13px | 13px | 400 |

规则：

1. 字间距保持 `0`。
2. 不用随视口宽度缩放字号。
3. 型号、参数值和状态标签需要可快速扫读。

### 3.3 间距与布局

采用 4px 基础栅格。

| Token | 值 | 用途 |
| --- | ---: | --- |
| `--space-1` | 4px | 图标与文字 |
| `--space-2` | 8px | 紧密元素 |
| `--space-3` | 12px | 表单内部 |
| `--space-4` | 16px | 卡片内边距 |
| `--space-6` | 24px | 模块间距 |
| `--space-8` | 32px | 页面分区 |
| `--space-12` | 48px | 大区块 |

容器：

| 设备 | 宽度 |
| --- | --- |
| PC | 最大 1280px |
| 宽屏 | 最大 1440px，内容不继续拉满 |
| 平板 | 24px 页面边距 |
| 手机 | 16px 页面边距 |

### 3.4 圆角与阴影

| 元素 | 圆角 |
| --- | ---: |
| 图标按钮、标签 | 4px |
| 输入框、下拉框 | 6px |
| 产品卡片、资料卡片 | 8px |
| 弹窗、抽屉 | 10px |

阴影只用于浮层：

```css
--shadow-card: 0 1px 2px rgba(15, 23, 42, 0.06);
--shadow-popover: 0 12px 32px rgba(15, 23, 42, 0.18);
```

禁止用大阴影让普通页面区块漂浮。

## 4. 全局布局

### 4.1 PC 布局

```text
Header: Logo / 主导航 / AI 入口 / 登录状态
Main: 根据页面使用双栏、三栏或表格布局
Footer: 公司、隐私、联系信息
```

Header 高度 64px，固定在顶部时必须有细边框或半透明磨砂背景，不能遮挡内容。

### 4.2 移动端布局

```text
Header: Logo / 菜单按钮 / 用户入口
Main: 单列内容
Bottom Nav: 首页 / 选型 / AI / 我的
```

底部导航高度 56px，页面底部必须预留安全区，不能遮挡按钮、PDF 页码或输入框。

## 5. 页面设计

### 5.1 首页

目标：用户 5 秒内完成关键参数输入或进入完整选型页。

首屏结构：

```text
深色品牌区
├── 左侧：标题、说明、可信信息
├── 右侧：快速选型面板
└── 下沿：热门应用和资料入口露出
```

首屏要求：

1. H1 使用“芯茂微智能选型平台”。
2. 快速选型面板包含输入电压范围、输出电压、输出电流和应用类型；认证需求为可选项，与 PRD 必填规则保持一致。
3. 面板按钮文案用“开始选型”，不使用“立即体验”等泛化营销词。
4. 首屏底部必须露出下一块内容，避免像封面页。
5. 首屏需出现可辨识的芯片实物、封装或应用板卡素材，主体清晰且不能被深色遮罩掩盖；不得使用无关科技插画或纯装饰纹理承担主视觉。

首页下方模块：

| 模块 | 目的 |
| --- | --- |
| 热门应用 | 让客户按应用进入选型 |
| 推荐型号 | 展示真实型号、参数、资料完整度 |
| 方案资料 | 展示可预览资料类型 |
| AI 技术助手 | 引导登录后提问 |

### 5.2 产品选型页

PC 使用左侧筛选 + 右侧结果。移动端使用顶部摘要 + 底部抽屉筛选。

```text
Selection Page
├── 筛选栏 320px
│   ├── 已选条件
│   ├── 电气参数
│   ├── 性能参数
│   ├── 合规参数
│   └── 固定底部操作
└── 结果区
    ├── 查询摘要
    ├── 排序和视图切换
    └── 推荐结果列表
```

推荐卡片必须包含：

1. 型号和系列。
2. 匹配等级：精确、近似、兜底。
3. 综合分。
4. 关键参数：输入电压、输出电压、输出电流、待机功耗、认证。
5. 推荐理由，最多 3 条。
6. 差异点，近似和兜底必须展示。
7. 规格书、方案资料、加入对比按钮。

卡片布局示例：

```text
┌──────────────────────────────────────────────┐
│ LP3798ESM                     精确匹配 96分  │
│ 高性能原边反馈系列                            │
├───────────────┬───────────────┬──────────────┤
│ 输入 85-265V  │ 输出 12V/2A   │ 待机 <0.075W │
├──────────────────────────────────────────────┤
│ 推荐理由                                      │
│ 集成 650V MOS / 满足七级能效 / 资料完整        │
├──────────────────────────────────────────────┤
│ [规格书] [方案资料] [加入对比]                │
└──────────────────────────────────────────────┘
```

交互要求：

- 筛选改动后不立即抖动刷新，点击“开始选型”或使用 500ms 防抖。
- 已选条件可单独移除。
- 空结果展示参数建议，不只显示“暂无数据”。
- 加入对比后固定底部对比栏，最多 3 个型号。

### 5.3 产品详情页

详情页以工程师扫读为目标。

页面结构：

```text
产品头部
├── 型号、系列、状态、资料完整度
├── 关键参数矩阵
├── 推荐应用
├── 规格书入口
└── 关联方案列表
```

关键参数矩阵建议使用 2-4 列，不用长段文字。资料入口要显示权限状态：可预览、登录后下载、资料整理中。

### 5.4 方案资料页

资料页要避免像普通附件列表，应像工程资料工作台。

PC 布局：

```text
左侧：资料目录和权限状态
中间：PDF/图片/表格预览区
右侧：方案摘要、关联型号、下载操作
```

移动端：

```text
顶部：方案摘要
中部：资料 Tabs
底部：预览和固定操作栏
```

PDF 预览规则：

1. 页码、缩放、下载按钮固定在预览区顶部。
2. 匿名用户第 4 页起显示清晰的权限遮罩。
3. 遮罩文案直接说明“注册后查看完整测试报告和下载方案资料”。
4. 登录成功后回到原页码。
5. PDF 加载失败必须提供重试和联系入口。
6. 匿名用户看到的第 4 页遮罩只是交互提示，实际请求必须来自仅含前 3 页的派生预览件；设计验收需配合接口越权测试，不能把遮罩视为安全边界。

### 5.5 AI 问答页

AI 页面必须体现“基于资料回答”，不能像普通聊天机器人。

布局：

```text
左侧：会话历史和推荐问题
中间：对话流
右侧：本轮引用来源 / 可理解的证据关系
底部：输入框
```

移动端隐藏左右侧栏，用抽屉展示历史和来源。

消息要求：

1. 用户消息右侧，AI 消息左侧。
2. AI 消息底部必须展示来源卡片。
3. 来源卡片显示文档名、页码、片段摘要。
4. 多跳回答以“相关型号/参数 -> 关联事项 -> 来源片段”展示可理解证据，不直接把 `event`、`entity`、向量分数等内部字段暴露给普通用户。
5. 低置信拒答用中性提示，不使用错误红。
6. 回答生成中显示逐字输出和来源准备状态。
7. 管理员调试视图可展开检索 trace，展示召回模式、耗时、命中 event/entity 和重排结果。
8. 来源无访问权限或资料已下架时不得渲染标题与片段；后端应已完成过滤，前端仍需按错误状态安全降级。

输入框要求：

- Enter 发送，Shift + Enter 换行。
- 空输入禁用发送。
- 输入框上方可展示 3 个真实业务问题建议。
- 发送失败保留原问题并允许重试。

### 5.6 登录与注册

登录注册可以是独立页，也可以从资料解锁弹窗进入。视觉要简洁，不做大幅营销插图。

必须包含：

1. 邮箱。
2. 密码。
3. 密码强度提示。
4. 隐私政策勾选。
5. 登录/注册切换。
6. 返回原页面能力。

资料解锁弹窗文案强调价值，不制造焦虑：

```text
注册后可查看完整测试报告、下载 BOM/原理图，并继续使用 AI 技术问答。
```

### 5.7 运营后台

后台是高频管理工具，不使用营销式卡片堆叠。

布局：

```text
固定侧栏 240px
顶部栏 56px
主内容区
  ├── 标题和说明
  ├── 筛选工具条
  ├── 表格
  └── 分页
```

表格要求：

1. 支持搜索、筛选、分页、排序。
2. 操作列固定在右侧。
3. 状态用标签，不用纯文字。
4. 删除、下架、重建索引等高风险操作必须二次确认。
5. 移动端后台可降级为只读或卡片列表，但核心操作不能错位。

知识库管理必须展示 `UPLOADED`、`PROCESSING`、`READY`、`FAILED` 状态、当前索引版本、更新时间和失败原因。重建索引期间保留当前可用版本；重试与重建操作必须防止重复点击，并在任务已存在时展示任务状态而不是重复创建。

## 6. 组件规范

### 6.1 按钮

| 类型 | 用途 |
| --- | --- |
| Primary | 提交、开始选型、确认保存 |
| Secondary | 返回、取消、查看详情 |
| Ghost | 深色背景上的次要操作 |
| Text | 表格内轻量操作 |
| Danger | 删除、下架、禁用 |

所有按钮必须有 default、hover、active、focus、disabled、loading 状态。

### 6.2 表单

1. Label 永远可见，不用仅 placeholder 作为字段名。
2. 单位放在输入框后缀，例如 `V`、`A`、`W`。
3. 错误提示紧贴字段下方。
4. 数值字段限制非法字符。
5. 移动端输入控件高度不低于 44px。

### 6.3 标签

| 状态 | 颜色 |
| --- | --- |
| 精确匹配 | 绿色 |
| 近似匹配 | 琥珀色 |
| 兜底推荐 | 灰蓝色 |
| 上架 | 绿色 |
| 草稿 | 灰色 |
| 下架 | 红色 |
| 已索引 | 蓝色 |

### 6.4 空状态

空状态必须给下一步动作。

| 场景 | 文案方向 | 动作 |
| --- | --- | --- |
| 无推荐 | 当前参数无完全匹配 | 调整参数 |
| 无资料 | 资料整理中 | 联系销售/返回详情 |
| 无线索 | 暂无客户行为 | 查看采集规则 |
| 无知识库 | 还未上传资料 | 上传文档 |

### 6.5 图标

统一使用已锁定的 Ant Design Icons。图标风格必须线性、统一描边，不手绘 SVG，不混用彩色 emoji 作为正式 UI 图标；只有不熟悉的纯图标按钮需要 Tooltip。

### 6.6 Ant Design 与 Tailwind 边界

1. Ant Design 负责表单、表格、弹窗、抽屉、分页、通知和可访问交互状态，并通过 `ConfigProvider` 统一主题 token。
2. Tailwind 只负责页面布局、间距、响应式和少量品牌外观，不使用全局选择器覆盖 Ant Design 内部 DOM。
3. 同一视觉属性只由一套系统控制，禁止在组件 `style`、Tailwind 类和全局 CSS 中重复声明。
4. 前台品牌页面允许封装业务组件；后台优先复用 Ant Design，不复制一套相似基础组件。

## 7. 响应式规则

| 断点 | 布局 |
| --- | --- |
| `>= 1200px` | PC 完整多栏 |
| `768px - 1199px` | 双栏或折叠侧栏 |
| `< 768px` | 单列、抽屉筛选、底部导航 |

硬性要求：

1. 移动端无横向滚动。
2. 表格在移动端改为卡片或关键字段列表。
3. 筛选抽屉底部操作按钮固定。
4. AI 输入框不能被底部导航遮挡。
5. PDF 工具栏在小屏下可横向内部滚动，但页面整体不能横向滚动。

## 8. 可访问性

1. 文本对比度至少满足 WCAG AA。
2. 所有表单控件有可读 label。
3. 键盘可访问主要操作。
4. 焦点状态清晰。
5. 错误信息不能只靠颜色表达。
6. PDF 页码、下载、关闭弹窗等按钮必须有 `aria-label`。

## 10. 视觉验收清单

每个页面交付时必须检查：

- [ ] 首屏能看出产品名称、用途和核心操作。
- [ ] 页面没有无意义大图、装饰渐变、空泛口号堆叠。
- [ ] 关键业务数据真实存在：型号、参数、资料、状态、来源。
- [ ] PC、平板、手机三种视口无布局错位。
- [ ] 文字不溢出按钮、标签、卡片和表格单元格。
- [ ] 加载、空状态、错误、无权限状态都已设计和实现。
- [ ] 所有按钮有 hover、focus、disabled、loading 状态。
- [ ] 移动端底部导航和键盘不遮挡关键操作。
- [ ] 推荐结果、资料页、AI 来源卡片具备明显信息层级。
- [ ] 后台页面能支持批量数据，不只适合 3 条演示数据。
- [ ] 使用至少 30 条产品、100 条资料、50 条线索和长中文/英文型号数据走查，无截断、跳动或异常空白。
- [ ] 在 `1440x900`、`1024x768`、`390x844` 三个固定视口完成截图对比；核心流程同时完成键盘与触屏走查。
- [ ] 浏览器控制台无错误，图片、PDF、来源链接和失败重试均使用真实接口验证。

## 11. 反模板检查

出现以下任一情况，不允许通过 UI 验收：

1. 首屏只有大标题、副标题和一个按钮。
2. 页面主要内容由低信息密度大卡片堆叠。
3. 使用大面积紫蓝渐变、发光圆球、无意义科技背景线。
4. 产品卡片没有关键参数和匹配原因。
5. AI 回答没有来源区域。
6. 资料页只是附件列表，没有预览和权限状态。
7. 后台像官网区块，不像可长期使用的数据管理工具。
8. 移动端只能浏览，不能完成核心操作。
